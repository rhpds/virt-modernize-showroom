== Proposed Timing

**Opening (5 minutes):**

* Introduce TravelCorp use case and current challenges
* Show end result: unified VM and container management
* Set the business context for modernization without rewrites

**Technical Demo (30–40 minutes):**

* Follow the modular approach based on audience
* Select 2-4 modules depending on time and customer priorities
* Focus on business value and operational efficiency
* Each module includes **Know** section for context and **Show** section for demonstration

**Closing (5–10 minutes):**

* Summarize how OpenShift Virtualization solves VM modernization challenges
* Connect demo scenarios to customer's real environment
* Allow time for Q&A and next steps discussion

== Troubleshooting Common Issues

*VM startup failures:*

* Check OpenShift Virtualization operator status
* Verify sufficient cluster resources (CPU, memory)
* Confirm VM image accessibility and permissions

*Service Mesh connectivity issues:*

* Validate Istio sidecar injection for VMs
* Check ServiceMesh member configuration
* Verify network policies and security contexts

*GitOps synchronization problems:*

* Confirm Git repository connectivity
* Check ArgoCD application health status
* Validate YAML syntax in Git configurations

*Performance or monitoring gaps:*

* Verify Prometheus scraping configuration
* Check Grafana data source connectivity
* Confirm Jaeger trace collection setup

*Authentication errors:*

* Validate OpenShift user permissions
* Check service account configurations
* Verify RBAC policy assignments
